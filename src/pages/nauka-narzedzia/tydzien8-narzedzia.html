WTF: Webpack

Webpack jest takim ogarniaczem. Odpowiada, za wykonywanie automatycznie części zadań w projekcie (np. importuje moduły niezależnie czy z mojego kodu czy z paczek npm'owych, konwertuje style z scss do css, ogarnia grafikę itd.)

WEBPACK
  • podstawowe narzędzie frontendowca
  • może automatyzować procesy
  • ułatwia development
  • używają go wiodące frameworki i biblioteki
  • warto znać podstawy
Z czego składa się konfiguracja webpacka?
  • Webpack'a możemy zainstalować za pomocą narzędzi npm'owych: mkdir webpack-demo
    cd webpack-demo
    npm init -y
    npm install webpack webpack-cli --save-dev
  • Konfigurujemy wejście (entry). Plikiem wejściowym domyślnie dla webpack'a jest: ./src/index.js,
  • konfigurujemy wyjście (output), plik ten umieszczany jest domyślnie w ./dist/main.js 
  • Domyślnie zdefiniowane są loader'y które pozwalają na operacje na plikach innych niż JS
  • na końcu definiujemy pluginy, które pozwalają na wykonywanie różnych operacji
  • Tryby pracy webpacka (mode):
    • development
    • production
  • możliwości webpacka rozszerzamy za pomocą paczek z NPM

WTF: Webpack Starterkit

Jest to szablon, z którego możemy skorzystać, kiedy będziemy robić swoje nowe projekty.

Co mamy w środku?
  • serwer lokalny z autoodświeżaniem
  • wsparcie dla scss i sourcemap
  • minifikacja
  • babel - pozwala, na działanie nowych rzeczy wprowadzonych w JavaScript'cie w różnych przeglądarkach (nie koniecznie wspierające te nowe standardy)
  • publikacja na github pages
Zaczynamy
  • Na początek zabawy z Starterkitem to zakładamy nowe repozytorium na githubie i pobieramy je do siebie do katalogu dev/. Skonfigurowanego webpacka przez Maćka Korsana możemy pobrać ze strony https://github.com/maciejkorsan/wtf-webpack-starter/generate git clone https://github.com/adresrepo
  • Odpalamy npm install  (tylko raz na projekt i w momencie gdy ściągamy nową wersję lub dokładamy coś od siebie)
  • Żeby uruchomić webpacka, to wpisujemy npm run start, za każdym razem jak zaczynamy pracę
  • localhost:8080
  • ctrl + c wyjście z działającego webpacka
  • katalog SRC to źródło naszej strony tutaj pracujemy
  • Ścieżki podajemy względem plików źródłowych (nie serwerowych!)

WTF: Webpack Starterkit - dodawanie nowych plików, sourcemapy

Dodajemy nowe pliki

Żeby webpack mógł pracować na nowych plikach trzeba je dodać do jego configa (webpack.common.js). Kopiujemy fragment kodu:

new HtmlWebpackPlugin({
template: "./src/pages/index.html",
inject: true,
chunks: ["index"],
filename: "index.html",
}),

I zmieniamy go tak, żeby odnosił się do naszego nowego pliku:

new HtmlWebpackPlugin({
template: "./src/pages/pawel.html",
inject: true,
chunks: ["index"],
filename: "pawel.html",
}),

W sekcji chunks definiujemy jakie pliki .js mają być załadowane w naszym projekcie. (pliki te muszą tyć zdefiniowane w sekcji entry (np. `index: "./src/js/index.js",)

W filename definiujemy nazwę pliku pod jaką ma być opublikowany, a ścieżkę do niego podajemy w template

Fajną rzeczą w webpacku jest to, że nie musimy dodawać tagów do pliku html ze skryptami i stylami. automatycznie doda to webpack do dokumentu.

Aby style były załadowane do naszych dokumentów musimy w je zaimportować przez plik .js. import '../scss/main.scss';
