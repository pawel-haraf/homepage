JavaScript: Obiekty

Prawie wszystko w JS jest obiektem ( nawet typy proste, mimo iż nie są obiektem - tak się zachowują). Obiekt jest strukturą danych. Dobrze nadaje się do odzwierciedlenia struktury otaczającego nas świata, który również jest obiektowy.

const deathStar = {
diameter: 120000,
fire: (target) => {
console.log(`${target} destroyed 💥`)
},
isOperating: true,
levels: 357,
name: 'Death Star',
population: 10000,
}

Każdy obiekt składa się z własności. Te własności tworzy para klucz: wartość. Wartości klucza mogą to być dowolne typy proste (stringi, liczby, boolean,  itd.). Kluczem może być nawet funkcja.

Klucz jest unikany w ramach obiektu, a wartości to mogą być prymitywy, funkcje i zagnieżdżenia np.:

commander: { name: 'Darth Vader', age: 44, },

W obiektach zmienne nazywamy własnościami (property), a funkcje to metody (methods).

Dot notation ( . )

Wypisanie tylko jednej własności obiektu możliwe jest dzięki dot notation (".") np.

console.log(deathStar.name); // Death Star

Metody wywołujemy poprzez podanie nazwy obiektu, oraz tej metody:

deathStar.fire('Rebel ship') // Rebel ship destroyed 💥
Bracket notation []

Zamiast użycia kropek odwołujemy się do własności za pomocą nawiasu kwadratowego.

console.log(deathStar['name']); // Death Star

Musimy w tym przypadku pamiętać, że nazwę własności musimy podać jako string (użyć apostrofów ' '). W innym wypadku dostaniemy undefinded. Chyba, że zdefiniujemy nazwę tej własności w zmiennej jako string.

console.log(deathStar[name]); // undefinded
const myProperty = 'name';
console.log(deathStar[myProperty]); // Death Star

const showMeProperty = (myProperty) => {
console.log(`Twoja własność ${myProperty} to: ${deathStar[myProperty]}`); // Twoja własność level to: 357
}
showMeProperty('levels')

Immutability w obiektach

Nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi, a zmiennymi. Czyli jak mamy sytuacje jak poniżej:

const numberOne = 42;
const numberTwo = 42;
const nameOne = 'Maciek';
let nameTwo = 'Stefan';

nameTwo = 'Maciek';

To znaczy, że my zmieniamy połączenie zmiennej nameTwo z 'Stefan' na 'Maciek'  .  Wartość 'Maciek' cały czas będzie istnieć. Jej nie usuwamy z wszechświata wartości😛

Jak immutability zachowuje się w obiektach

Mamy dwa obiekty:

const humanOne = {
name: 'Maciek',
age: 32
}

const humanTwo = {
name: 'Stefan',
age: 32
}

humanTwo.age = 35

console.log(humanTwo.age); // 35

Zmiana połączenia jest możliwa, ponieważ tylko połączenie między zmienną, obiektem jest sztywne, niezmienialne. Za to już można zmieniać połączenia poszczególnych własności obiektu, a zmiennymi.

Jest to tzw. mutowalność obiektów

A co jeśli mamy poniższy przypadek:

const humanOne = {
name: 'Maciek',
age: 32
}

const humanTwo = humanOne;
console.log(humanOne); // { name: 'Maciek', age: 32 }
console.log(humanTwo); // { name: 'Maciek', age: 32 }

humanTwo.age = 35;
console.log(humanOne); // { name: 'Maciek', age: 35 }
console.log(humanTwo); // { name: 'Maciek', age: 35 }

Spowoduje to, że podepniemy się zmienną humanTwo pod obiekt który jest podłączony pod zmienną humanOne .  Jeśli zmienimy własność humanTwo.age = 35;, to zmienimy  własność age w obiekcie do którego są podpięte zmienne humanOne i humanTwo . Ponieważ jest to ten sam obiekt.

Dlatego trzeba pamiętać, że w JS mutowalność opiera się przede wszystkim na połączeniach między obiektami i zmiennymi. Od tego później będą zależeć wszystkie zmiany.

Kolejny przypadek teoretyczny typu co się stanie gdy?😜
const humanOne = {
name: 'Maciek',
age: 32
}

const humanTwo = {
name: 'Stefan',
age: humanOne.age
}

console.log(humanTwo); // { name: 'Maciek', age: 32 }

Własność age: humanOne  obiektu humanTwo spowoduje wykonanie połączenia do wartości (w tym przypadku 32), a nie pod klucz ('age'). Czyli jak zmienimy własność age dla obiektu humanOne  to otrzymamy wynik jak w przypadku poniżej:

humanOne.age = 35;
console.log(humanOne) // {name: "Maciek", age: 35}
console.log(humanTwo); // {name: "Stefan", age: 32}
Zagnieżdżanie
const humanOne = {
name: 'Maciek',
age: 32
address {
street: 'Warszawska'
city: "Białystok'
}

const humanTwo = {
name: 'Stefan',
age: 35
address: humanOne.address
}

W powyższym przypadku,  nie tworzymy nowego obiektu, tylko podpinamy się pod obiekt zagnieżdżony w własności address  obiektu humanOne . Czyli wszystkie zmiany w obiekcie w własności addres  obiektu humanOne nie będą zmieniać połączenia  i będą tożsame dla obiektu humanOne  i humanTwo .

const humanOne = {
name: 'Maciek',
age: 32,
address: {
street: 'Warszawska',
city: 'Białystok'
}
}
const humanTwo = {
name: 'Stefan',
age: 35,
address: humanOne.address
}

console.log(humanOne); // { name: "Maciek", age: 32 adress: { street: "Warszawska", city: "Białystok" }
} console.log(humanTwo); //{ name: "Stefan", age: 32, address: { street: "Warszawska", city: "Białystok" } }

humanOne.address.street = 'Lipowa';

console.log(humanOne); // { name: "Maciek", age: 32 adress: { street: "Lipowa", city: "Białystok" }
}
console.log(humanTwo); //{ name: "Stefan", age: 32, address: { street: "Lipowa", city: "Białystok" } }

Żeby tego uniknąć możemy zastosować zapis:

const humanOne = {
name: 'Maciek',
age: 32,
address: {
street: 'Warszawska',
city: 'Białystok'
}
}
const humanTwo = {
name: 'Stefan',
age: 35,
address: {
street: humanOne.address.street,
city: 'Białystok'
}
}

console.log(humanOne); // { name: "Maciek", age: 32 adress: { street: "Warszawska", city: "Białystok" }
}
console.log(humanTwo); //{ name: "Stefan", age: 32, address: { street: "Warszawska", city: "Białystok" } }

humanOne.address.street = 'Lipowa';

console.log(humanOne); // { name: "Maciek", age: 32 adress: { street: "Lipowa", city: "Białystok" }
}
console.log(humanTwo); //{ name: "Stefan", age: 32, address: { street: "Warszawska", city: "Białystok" } }

Co spowoduje, że powstaną dwa niezależne obiekty o nazwie address  dla każdego z obiektów.  Zmiana własności street dla jednego z obiektów, nie spowoduje zmiany własności w drugim obiekcie.
