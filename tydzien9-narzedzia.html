GIT:Branche

Czym jest branch?

Branche:

  • pozwalają na bezkarne testowanie
  • służą do bardziej efektywnego rozwijania oprogramowania
  • pozwalają zrównoleglić pracę
  • zwiększają czytelność postępu prac nad projektem
  • ułatwiają wypuszczanie rzeczy na świat

Branche umożliwiają bezkarne pracowanie nad daną wersją projektu i sprawdzenie jak one będą się zachowywać w projekcie. Po czym można je wdrożyć do głównej wersji projektu.

Podstawowym branchem w git'ie jest master.

  • git branch - pokazuje na jakim branchu (gałęzi) aktualnie pracujemy
  • git branch nazwabrancha - polecenie utworzy gałąź (branch) o nazwie nazwabrancha
  • git checkout nazwabrancha - przełączamy się  na branche "nazwabrancha"
  • git push --set-upstream origin nazwabrancha - polecenie pozwala dodać nowo powstałą gałąź na serwer zewnętrzny (np. na GitHub'a)
  • git merge nazwabrancha - polecenie sprawia, że zmiany z innej gałęzi zostają wprowadzone do gałęzi master (trzeba tylko pamiętać, żeby przełączyć się na gałąź master)

Markdown

Czym jest MARKDOWN?

  • Uniwersalne formatowanie tekstu
  • Czytelne zarówno w formie źródła, jak i wyniku
  • Bezpieczne wprowadzanie treści bez konieczności korzystania z HTML
  • Nie trzeba pisać edytora

Formatowanie tekstu uzyskujemy poprzez zastosowanie odpowiednio stworzonego języka znaczników, dzięki któremu tekst jest od razu zamieniany na odpowiedni format tekstu.

Nagłówki
  • # Nagłówek (H1)
  • ## Nagłówek II (H2)
  • ### Nagłówek III (H3)
  • ...

Czy frontendowiec musi być grafikiem?

Nie, ale powinien znać kilka podstawowych zasad, które trzeba wziąć pod uwagę, przy projektowaniu stron internetowych.

Kilka zasad "bieda projektowania"
  1. Zacznij od prawdziwej treści!
  2. Naszkicuj wstępny układ na papierze
  3. Porzuć wizję strony idealnej, zacznij od najniższego poziomu, który daje jakąś wartość
  4. Działaj "krok po kroku" (one step a time)
  5. Trzymaj się siatki / grida
  6. Zacznij od czarnego / białego / odcieni szarości
  7. Kolory zostaw na koniec i ogranicz ich liczbę do minimum
  8. Nie wymyślaj koła na nowo
  9. Wykorzystaj gotowe elementy / znajdź UI KIT
  10. Kopiuj!
